# 📌 링크 노트 MVP 기획안 (정리본)

## 1. 목표

> "링크를 보내면, 자동으로 저장 + 요약 + 분류해주는 개인용 노트 앱"

초기 타겟:
**유튜브 링크 → 자동 저장 + 한 줄 요약**

---

## 2. MVP 범위 (명확화)

### 2-1. 수집
- 유튜브 링크를 **공유 버튼 → 앱 선택**으로 전달
- 앱이 URL 자동 인식 & 저장

> ※ 인스타, 스레드, 웹은 2차 확장

---

### 2-2. 처리
- 링크 메타데이터 수집 (제목, 썸네일)
- AI로 **한 줄 요약 생성**

예:
> "이 링크는 '유튜브 수익화 전략'과 관련된 콘텐츠입니다."

---

### 2-3. 정리
- 기본 분류:
  - 자동 카테고리 (예: 투자 / 건강 / 개발 / 기타)
- 태그 3~5개 자동 생성

---

### 2-4. 조회
- 저장된 링크 카드 리스트
- 클릭 시:
  - 썸네일
  - 제목
  - 한 줄 요약
  - 태그

---

### 2-5. 유료화 고려
MVP에는 **결제 X**,
하지만 구조는 **유료 확장 가능**하게 설계.

---

## 3. 단계별 구현 로드맵

---

# 🟦 PHASE 1 — 핵심 MVP (배포 가능한 최소 단위)

> "유튜브 링크 저장 + 한 줄 요약"

### 기능 스펙

| 구분 | 구현 항목 |
|------|----------|
| 앱 | 공유 인텐트 수신 |
| 서버 | URL 저장 API |
| 처리 | 메타데이터 파싱 |
| AI | 한 줄 요약 생성 |
| DB | 링크 + 요약 저장 |
| UI | 카드 리스트 |

---

### 개발 단위로 쪼개면

### 1️⃣ 앱 레벨
- [ ] 공유 인텐트 수신
- [ ] URL 파싱
- [ ] 서버로 POST 요청
- [ ] 저장 완료 화면

---

### 2️⃣ 서버 레벨 (FastAPI)
- [ ] `/save-link` API
- [ ] 유튜브 메타데이터 추출
- [ ] AI 요약 호출
- [ ] DB 저장

---

### 3️⃣ AI 처리

입력:
```
제목 + 설명
```

출력:
```
한 줄 요약
```

---

### 4️⃣ DB 구조 (초기)

| 필드 | 설명 |
|------|------|
| id | 링크 ID |
| url | 원본 URL |
| title | 제목 |
| summary | 한 줄 요약 |
| tags | 자동 태그 |
| created_at | 저장일 |

---

### 5️⃣ UI
- 홈: 저장된 링크 카드
- 상세: 요약 + 태그

---

### PHASE 1 성공 기준
- 링크 저장 성공률 95%+
- 요약 생성 실패율 < 5%
- 사용자 본인 만족도
